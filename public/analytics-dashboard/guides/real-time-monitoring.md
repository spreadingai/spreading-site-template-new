# 实时监控

- - -

实时监控分别从业务视角和服务视角为您提供音视频服务的最新实时数据，数据更新频率可达到分钟级，助您有效监控业务进行中的状态变化，及时发现问题解决问题，保证业务稳定。

<Warning title="注意">

实时监控为您提供最近 2 小时内的实时数据，每分钟更新 1 次。若您需要查看历史数据，可前往 [运营大盘](/analytics-dashboard/guides/operations-dashboard/function-introduction)。  
</Warning>

## 访问步骤

<Warning title="注意">

联系 ZEGO 售前工程师或销售人员开通“实时监控”。
</Warning>

1. 登录 [ZEGO 控制台](https://console.zego.im)，从左侧导航栏进入星图；
2. 在导航栏点击“实时监控”，展开其子页面；
3. 单击“业务监控”、“服务监控”任意按钮，进入页面；
4. 选择 AppID 和想要查询的时间范围，点击查询，即可查看相关数据。

### 数据说明

| 名称 | 含义 |
| -- | -- |
|在线用户数| 统计当前时刻同时在线的用户总数。 |
|房间并发数| 统计当前时刻同时存活的房间数。  |
|最大房间用户数| 当前时刻所有存活的房间中，在线用户最多的房间及其对应的用户数。 |
|视频流畅度| 评价整体通话视频流畅情况，以每次通话的视频卡顿率进行换算取值，流畅度越高，表示用户观看视频的体验越好；流畅度=1-卡顿率。 |
|音频流畅度| 评价整体通话音频流畅情况，以每次通话的音频卡顿率进行换算取值，流畅度越高，表示用户收听音频的体验越好；流畅度=1-卡顿率。 |
|用户音频体验| 用于评价用户听到音频的感官体验，分为优、较优、良好、差4个等级，各个等级对应的体验如下：<ul><li>优：音质佳，清晰流畅。</li><li>较优：音质较好，偶有音质损伤，但依然清晰。</li><li>良好：音质一般，不是非常流畅，需要集中注意力。</li><li>差：音质很差，有杂音及语义丢失，难以交流。</li></ul> |
|进房耗时| 评价用户登录房间的等待情况，以请求登录到成功登录房间的间隔耗时进行取值，进房耗时越低，表示用户登录房间等待的时间越低，登录体验越好。|
|首帧耗时| 评价用户看到第一帧画面或听到第一帧声音的等待情况，以请求拉流到完成第一帧渲染的间隔耗时进行取值，首帧耗时越低，表示用户拉流等待的时间越短。|
|端到端延迟| 评价发送端到接收端的延迟情况，通过计算推流地区的上行延迟、服务间的延迟、拉流地区的下行延迟以及本地渲染的耗时得到端到端延迟。|
|上行延迟| 评价推流用户客户端到服务器的延迟情况。|
|下行延迟| 评价服务器到拉流用户客户端的延迟情况。|
|上行丢包率| 评价推流用户客户端到服务器的丢包情况。|
|下行丢包率| 评价服务器到拉流用户客户端的丢包情况。|
|端到端丢包率| 评价发送端到接收端的丢包情况，通过计算推流地区的上行丢包率、服务间的丢包率、拉流地区的下行丢包率得到端到端丢包率。|

## 服务监控

您可以从“服务监控页面”及时了解房间服务、推流服务、拉流服务的可用性，第一时间判断是否出现影响大面积用户的服务异常，并及时跟进解决，保障业务稳定。

### 数据说明
| 名称 | 含义 |
| -- | -- |
|房间并发数| 统计当前时刻同时存活的房间数。 |
|房间登录成功率| 评价客户端正常发起登录请求，到正常登录房间的情况，观察该指标可跟踪了解是否出现大面积影响用户正常登录的故障发生。  |
|推流并发数| 统计当前时刻同时推流的数量。 |
|推流成功率| 评价客户端到服务器进行推流请求，正常联通的情况，观察该指标可跟踪是否出现大面积影响用户正常使用的异常发生。|
|拉流并发数| 统计当前时刻同时拉流的数量。|
|拉流成功率| 评价客户端到服务器进行拉流请求，正常联通的情况，观察该指标可跟踪是否出现大面积影响用户正常使用的异常发生。|
