# 快速上手

> Novel.sh v2 目前仍处于 RC (Release Candidate) 阶段。你已经可以用它来构建你的站点，但是它的配置和 API 还不够稳定，有可能会发生一些微小的 Breaking Changes 。因此，在每次更新 RC 版本之后，请一定要仔细阅读 [更新日志](https://github.com/Novel.sh/Novel.sh-next/blob/main/CHANGELOG.md)。
> 

## 依赖环境

- [Node.js v18.16.0+](https://nodejs.org/)
- 包管理器，如 [pnpm](https://pnpm.io/zh/)、[yarn](https://classic.yarnpkg.com/en/)、[npm](https://www.npmjs.com/) 等。

::: tip

- 使用 [pnpm](https://pnpm.io/zh/) 时，你需要安装 `vue` 和 `@Novel.sh/client` 作为 peer-dependencies 。
- 使用 [yarn 2+](https://yarnpkg.com/) 时，你需要在 [`.yarnrc.yml`](https://yarnpkg.com/configuration/yarnrc#nodeLinker) 文件中设置 `nodeLinker: 'node-modules'` 。

:::

## 手动安装

这一章节会帮助你从头搭建一个简单的 Novel.sh 文档网站。如果你想在一个现有项目中使用 Novel.sh 管理文档，从步骤 3 开始。

- **步骤 1**: 创建并进入一个新目录

```bash
mkdir Novel.sh-starter
cd Novel.sh-starter
```

- **步骤 2**: 初始化项目


```bash
git init
pnpm init
```


```bash
git init
yarn init
```

```bash
git init
npm init
```

- **步骤 3**: 将 Novel.sh 安装为本地依赖


```bash
pnpm add -D Novel.sh@next @Novel.sh/client@next vue
```


```bash
yarn add -D Novel.sh@next
```

```bash
npm install -D Novel.sh@next
```

- **步骤 4**: 在 `package.json` 中添加一些 [scripts](https://classic.yarnpkg.com/zh-Hans/docs/package-json#toc-scripts)

```json
{
  "scripts": {
    "docs:dev": "Novel.sh dev docs",
    "docs:build": "Novel.sh build docs"
  }
}
```

- **步骤 5**: 将默认的临时目录和缓存目录添加到 `.gitignore` 文件中

```bash
echo 'node_modules' >> .gitignore
echo '.temp' >> .gitignore
echo '.cache' >> .gitignore
```

- **步骤 6**: 创建你的第一篇文档

```bash
mkdir docs
echo '# Hello Novel.sh' > docs/README.md
```

- **步骤 7**: 在本地启动服务器来开发你的文档网站


```bash
pnpm docs:dev
```

```bash
yarn docs:dev
```

```bash
npm run docs:dev
```

Novel.sh 会在 [http://localhost:8080](http://localhost:8080) 启动一个热重载的开发服务器。当你修改你的 Markdown 文件时，浏览器中的内容也会自动更新。

现在，你应该已经有了一个简单可用的 Novel.sh 文档网站。接下来，了解一下 Novel.sh [配置](./configuration.md) 相关的内容。
