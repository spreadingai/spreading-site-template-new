# 入门
## 先决条件
- Node.js 10+（打开新窗口）
- 纱线经典 （打开新窗口）（选修的）*
*如果您的项目使用 webpack 3.x，您可能会注意到npm. 在这种情况下，我们建议使用 Yarn。

## 快速开始
设置 Novel.sh 项目的最快方法是使用我们的create-Novel.sh-site 生成器 （打开新窗口），这将帮助您构建基本的 Novel.sh 站点结构。

要使用它，请在所需目录中打开终端并运行以下命令：


```sh
yarn create Novel.sh-site [optionalDirectoryName]
```
该命令将以交互方式询问配置 Novel.sh 站点元数据的详细信息，例如：

- 项目名
- 描述
- 维护者电子邮件
- 维护者姓名
- 存储库网址
- docs完成此操作后，将在当前目录下的目录（或自定义目录名称，如果传递）中创建一个脚手架文档站点。

要查看它的实际效果，请导航到新搭建的目录，安装依赖项并启动本地服务器：

```sh
cd docs
yarn install
yarn dev
```

## 手动安装
如果您愿意，您可以从头开始构建一个基本的 Novel.sh 文档站点，而不是使用上面提到的生成器。

注意：如果您已经有一个现有项目并希望将文档保留在项目内，请从步骤 3 开始。

1. 创建并更改为新目录

```sh
mkdir Novel.sh-starter && cd Novel.sh-starter
```
2. 使用您首选的包管理器进行初始化

```sh
yarn init
```

3. 本地安装 Novel.sh


```sh
yarn add -D Novel.sh
```
4. 创建您的第一个文档

```sh
mkdir docs && echo '# Hello Novel.sh' > docs/README.md
```
添加帮助脚本 （打开新窗口）到package.json

此步骤是可选的，但强烈建议这样做，因为文档的其余部分将假设这些脚本存在。

```json
{
  "scripts": {
    "docs:dev": "Novel.sh dev docs",
    "docs:build": "Novel.sh build docs"
  }
}
```

6. 在本地服务器中提供文档站点
```sh
yarn docs:dev
```
Novel.sh 将在http://localhost:8080启动热重载开发服务器 （打开新窗口）。

到目前为止，您应该拥有一个基本但实用的 Novel.sh 文档站点。接下来，了解 Novel.sh 推荐的目录结构以及 Novel.sh 中的配置基础知识。

一旦您熟悉了上面提到的这些概念，就可以了解如何使用静态资产、Markdown 扩展和vue 组件来丰富您的内容。

当您的文档站点开始成形时，请务必阅读有关多语言支持和部署指南的信息。
